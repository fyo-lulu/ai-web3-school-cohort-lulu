prompt：负责软性约束表达，模型硬约束靠系统。
instruction：给模型的任务规则，最好分目标/可用输入/禁止行为/输出格式失败格式
few-shot：测试示例
Structured output：结构化输出

Prompt context：内容注入让模型泄漏信息等

一个稳定的工具型 Agent 上下文，通常不只是“用户问题 + 一段 JSON”。它还应该包括：
- 当前任务状态
- 工具返回结果
- 相关日志或证据
- 可信数据来源
- 外部检查结果
- 用户原始意图
- 系统禁止事项和输出 schema

知识库至少要记录：
- 文档来源和 URL
- 最后更新时间
- 适用协议版本
- 适用版本或运行环境
- 是否来自官方或第三方
- 是否需要人工复核

一个靠谱的 Agent 上下文通常会分层：
- 指令层：系统规则、工具使用规则、禁止事项。
- 任务层：用户目标、本次会话参数。
- 事实层：链上状态、工具结果、simulation。
- 知识层：文档、标准、论坛、历史案例。
- 记忆层：用户偏好和项目配置。
